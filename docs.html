Documentation Introduction

Introduction

Welcome to the Adrenaline API documentation. This guide covers all available libraries, functions, and classes.

Adrenaline provides a comprehensive Lua scripting environment for Roblox with advanced execution capabilities, deep API access, and extensive library support.

Overview

Adrenaline's scripting API is designed to give developers and power users complete control over the Roblox environment. Whether you're building automation tools, testing security, or extending functionality, the Adrenaline API provides the primitives you need.

Note: This documentation is for Adrenaline API version 2.0+. Some functions may differ from earlier versions.

Getting Started

To begin scripting with Adrenaline, you'll need the executor injected into your Roblox client. Once injected, the global environment is extended with all Adrenaline-specific libraries.

Lua — Basic Script
-- Check if Adrenaline is loaded
if Adrenaline then
    print("Adrenaline API loaded successfully")
end

Cache Overview

The cache module gives you simple key-value storage that persists for the duration of the Adrenaline session.

Lua — Cache Example
-- Store a value
Cache.set("key", "value")

-- Retrieve a value
local val = Cache.get("key")

-- Check if key exists
if Cache.has("key") then
    print("Key exists")
end
Warning: Cached data is stored in memory only and will be cleared when the Adrenaline process exits.

Signals Overview

Signals allow different scripts to communicate by emitting and listening for named events.

Lua — Signal Example
-- Create a signal
local mySignal = Signal.new("MyEvent")

-- Connect a handler
mySignal:Connect(function(data)
    print("Received:", data)
end)

-- Fire the signal
mySignal:Fire("Hello World")

WebSocket Overview

Connect to WebSocket servers for real-time bidirectional communication directly from your scripts.

Lua — WebSocket Client

Frequently Asked Questions

How do I load a library?

All Adrenaline libraries are available globally once the executor is injected. Simply call any function by its library name, e.g. Cache.set("key", "value").

Can I use multiple libraries together?

Yes, all libraries are designed to work together seamlessly. You can mix and match functions from different libraries in the same script.

Is there a limit on cache size?

The cache library has a configurable maximum size. By default, it supports up to 1000 entries. Use Cache.setMaxSize(n) to adjust.

How do I report a bug?

Report bugs through our Discord server or GitHub repository. Include the library name, function, and a minimal reproduction script.
