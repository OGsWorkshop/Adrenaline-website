Documentation Introduction

Introduction

Welcome to the Adrenaline API documentation. This guide covers all available libraries, functions, and classes.

Adrenaline provides a comprehensive Lua scripting environment for Roblox with advanced execution capabilities, deep API access, and extensive library support.

Overview

Adrenaline's scripting API is designed to give developers and power users complete control over the Roblox environment. Whether you're building automation tools, testing security, or extending functionality, the Adrenaline API provides the primitives you need.

Note: This documentation is for Adrenaline API version 2.0+. Some functions may differ from earlier versions.

Getting Started

To begin scripting with Adrenaline, you'll need the executor injected into your Roblox client. Once injected, the global environment is extended with all Adrenaline-specific libraries.

Lua — Basic Script 
-- Check if Adrenaline is loaded
if Adrenaline then
    print("Adrenaline API loaded successfully")
end

The Adrenaline API is organized into modular libraries, each providing specific functionality. Libraries are accessible globally once the executor is injected.

Actors

The Actors library provides functions for managing Roblox Actor instances within parallel Lua scripts.

Parallel execution is essential for complex Roblox scripts. With Actors, tasks can be distributed over multiple cores to maximize efficiency without bottlenecking the main Lua thread.

Warning: Ensure clean task synchronization to prevent race conditions or unexpected parallel behaviors.

Cache

The Cache library provides in-memory caching utilities for storing and retrieving data across script executions.

The cache module gives you simple key-value storage that persists for the duration of the Adrenaline session.

Basic Usage

Below is an example of storing, checking, and retrieving items from the cache.

Lua — Cache Example 
-- Store a value
Cache.set("key", "value")

-- Retrieve a value
local val = Cache.get("key")

-- Check if key exists
if Cache.has("key") then
    print("Key exists")
end
Warning: Cached data is stored in memory only and will be cleared when the Adrenaline process exits.

Closures

The Closures library enables manipulation and introspection of Lua closures and functions.

Functions can be queried for their upvalues, constants, env references, and other internal debug information safely.

Console

The Console library provides enhanced console output and logging capabilities.

Use the console functions to customize execution feedback logs, error dumps, and tracing outputs.

Tip: Use Console.clear() to clear the output window at any time.

Crypt

The Crypt library provides cryptographic functions including hashing, encryption, and encoding.

Encrypt strings, generate MD5/SHA-256 hashes, or use standard Base64 encode and decode functions directly inside Adrenaline.

Debug

The Debug library provides debugging utilities and introspection tools for Lua scripts.

Inspect active stack frames, variables, hook internal script setups, and trace call structures dynamically.

Drawing

The Drawing library provides a comprehensive 2D rendering API for creating custom overlays and UI elements.

The Drawing library allows you to create and manipulate 2D shapes, text, and images rendered directly on screen, bypassing traditional Roblox UI structures.

Drawing Overview

Creating visual UI highlights, esp boxes, or customized text indicators with the drawing API is simple.

Lua — Drawing a Square 
local square = Drawing.new("Square")
square.Size = Vector2.new(200, 200)
square.Position = Vector2.new(100, 100)
square.Color = Color3.fromRGB(139, 92, 246)
square.Filled = true
square.Visible = true

Encoding

The Encoding library provides data encoding and serialization utilities.

Support for JSON serialization (HttpService:JSONEncode equivalents), URL encoding, binary transformations, and hex conversions are included.

Environment

The Environment library allows you to interact with and modify the Lua execution environment.

Access registries, alter environment tables, declare new globals, or isolate script environments completely.

Filesystem

The Filesystem library provides file and directory operations from within your scripts.

Read, write, and manage files on the local filesystem sandbox directly from your Adrenaline scripts.

Filesystem Overview

Operations are sandboxed to the Adrenaline workspace folder for safety and security.

Function Description
Filesystem.read(path)Read file contents as string
Filesystem.write(path, data)Write string data to a file
Filesystem.append(path, data)Append data to a file
Filesystem.exists(path)Check if a file exists
Filesystem.delete(path)Delete a file

Input

The Input library provides mouse and keyboard simulation capabilities.

Simulate keystrokes, mouse moves, clicks, and drag inputs programmatically directly through the executor system.

Instances

The Instances library extends Roblox Instance manipulation with additional capabilities.

Bypass properties, modify internal instance parameters, cache references, and access hidden core gui properties.

LuaStateProxy

LuaStateProxy provides low-level access to the underlying Lua state and execution environment.

Direct stack modifications and registry access are exposed here. Use with intense caution.

Caution: Improper use of LuaStateProxy can crash the executor or the Roblox client. Use with care.

Metatable

The Metatable library provides utilities for working with Lua metatables and metamethods.

Read-only metatables can be bypassed or hooked safely, allowing custom hooks on __index, __newindex, or __namecall.

Miscellaneous

The Miscellaneous library contains utility functions that don't fit into other categories.

Includes functions like setting clipboard text, fetching online contents, or handling system state calls.

Success: Most miscellaneous functions have no side effects and are safe to call at any time.

oth

The oth library provides miscellaneous Roblox-specific utility functions.

Control task yields, simulate Roblox network events, or hook basic physics behaviors.

RakNet

The RakNet library provides low-level networking utilities for intercepting and manipulating network traffic.

Intercept incoming/outgoing replication packets, edit physics coordinates mid-transit, or debug Roblox protocol data.

Reflection

The Reflection library enables runtime type introspection and reflection capabilities.

Retrieve class descriptors, method signatures, properties, and execute hidden functions from the Roblox reflection database.

Scripts

The Scripts library provides utilities for managing and executing Lua scripts.

Load local script modules, spawn concurrent script instances, or isolate specific code execution channels.

Signals

The Signals library provides an event-driven communication system between scripts.

Signals allow different scripts to communicate by emitting and listening for named events dynamically.

Signals Overview

Connect multiple event handlers or trigger script listeners on custom triggers easily.

Lua — Signal Example 
-- Create a signal
local mySignal = Signal.new("MyEvent")

-- Connect a handler
mySignal:Connect(function(data)
    print("Received:", data)
end)

-- Fire the signal
mySignal:Fire("Hello World")

VoltSignal

VoltSignal provides an advanced signal implementation with additional features over the standard Signal library.

VoltSignal is designed for enterprise systems requiring promise-based sequencing, filtering triggers, and auto-disconnection parameters.

Note: VoltSignal is an enhanced version of Signals with promise-like chaining and filtering capabilities.

WebSocket

The WebSocket library provides WebSocket client functionality for real-time communication.

Connect to WebSocket servers for real-time bidirectional communication directly from your scripts, bypass CORS limitations and fetch external resources.

WebSocket Overview

Start a websocket client, listen to inbound commands, and output values.

Lua — WebSocket Client 
local ws = WebSocket.connect("wss://example.com")

ws.OnMessage(function(msg)
    print("Received:", msg)
end)

ws:Send("Hello server!")

Frequently Asked Questions

Find answers to common questions about the Adrenaline executor and API integration.

How do I load a library?

All Adrenaline libraries are available globally once the executor is injected. Simply call any function by its library name, e.g. Cache.set("key", "value").

Can I use multiple libraries together?

Yes, all libraries are designed to work together seamlessly. You can mix and match functions from different libraries in the same script.

Is there a limit on cache size?

The cache library has a configurable maximum size. By default, it supports up to 1000 entries. Use Cache.setMaxSize(n) to adjust.

How do I report a bug?

Report bugs through our Discord server or GitHub repository. Include the library name, function, and a minimal reproduction script.
